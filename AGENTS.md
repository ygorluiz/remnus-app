# Project Details: Remnus

**Remnus** is a Notion-like application built around a **workspace** model. Users can create standalone pages (title + markdown) and customizable databases (dynamic columns, table/kanban/calendar views) ‚Äî both living side by side in a unified sidebar. Each database row is also a page with markdown content.

### Rules for All Future Development

**Every new component or server action that surfaces user-facing text MUST follow these rules:**

1. **Client components** ‚Äî `import { useTranslations } from 'next-intl'` and call `useTranslations('Namespace')` inside the component body.
2. **Server components / layouts** ‚Äî `import { getTranslations } from 'next-intl/server'` and `await getTranslations('Namespace')`.
3. **Server actions** ‚Äî same as above; use `getTranslations('Errors')` for error messages returned to the client.
4. **Add all new keys to ALL 8 files** before committing. Missing keys fall back to English (`en.json`) via the deep-merge in `request.ts`, then to the key name.
5. **No hardcoded display strings** ‚Äî not even English fallbacks like `|| 'Untitled'`. Always use `t('key')`.
6. **Date formatting** ‚Äî use `useLocale()` (client) or locale from `getRequestConfig` (server) instead of `'en-US'`.
7. **Namespace selection** ‚Äî pick the closest existing namespace. Create a new one only for a clearly standalone domain (add to all 8 files and document here).

### Agent Task Management & Work Plan

When working on project tasks, agents MUST use the **remnus-mcp** server to interact with the **Work Plan** database:
1. **Querying Tasks:** Retrieve pending tasks (e.g. Sprint-specific) from the "Work Plan" database using the `query_database` tool (optionally listing all rows and filtering in memory if database query filters fail).

### Auth System

- **Auth System:** Supports Google and GitHub OAuth providers. Email/password login and register pages have been removed.
- **Config split:** `src/auth.config.ts` (edge-safe, no DB, middleware only) + `src/auth.ts` (full, server components/actions).
- **Session access:** Always use `getCurrentUser()` from `src/lib/auth/session.ts` in server actions ‚Äî **never** `auth()` directly. It is `React.cache`-wrapped to run at most once per request.
- **createdAt gotcha:** DrizzleAdapter stores `CURRENT_TIMESTAMP` as text (breaks Drizzle timestamp parsing). All `workspaces` and `workspace_members` inserts pass explicit `createdAt: new Date()`. The `createUser` event also force-updates OAuth users immediately after row creation.
- **First-user bootstrap:** The very first non-demo user (any provider) is auto-promoted to `admin` and added as owner to all memberless workspaces.
- **Demo mode:** **per-visitor ephemeral accounts** ‚Äî each "Try the demo" click provisions a fresh throwaway user (`demo+<uuid>@remnus.com`, role `demo`) with its own freshly seeded workspace, so concurrent demo visitors never share or overwrite each other's data. `loginAsDemo()` ([demo.ts](src/lib/actions/demo.ts)): (1) if the caller already holds a live demo session with a workspace, redirects straight back into it (no new account / no reseed on refresh); (2) requires at least one real (non-demo) user to exist; (3) reaps stale demo accounts inline (`cleanupStaleDemoUsers` ‚Äî demo users older than `DEMO_USER_TTL_MS` = 6h, bounded to `DEMO_CLEANUP_LIMIT` = 25/login, best-effort; no cron infra). All demo detection elsewhere keys on `role === 'demo'` (never the email), so ephemeral emails are transparent to the banner, analytics-skip, consent, and admin panel. **Cleanup deletes the demo user's workspaces explicitly** (via `workspace_members`) since `workspaces` has no FK to the user; the user delete then cascades sessions/accounts/members.
- **Access control:** All actions call `assertWorkspaceAccess(workspaceId)` or `assertDatabaseAccess(databaseId)` before executing. Unauthorized ‚Üí throws; unauthenticated ‚Üí `redirect('/login')`.
- **Env vars:** `AUTH_SECRET`, `AUTH_GOOGLE_ID`, `AUTH_GOOGLE_SECRET`, `AUTH_GITHUB_ID`, `AUTH_GITHUB_SECRET`.

### Billing & Plan Limits

**Model:** The subscription belongs to the **billing owner** (a user), not a workspace. A workspace's limits come from its `workspaces.billing_owner_id`'s plan (`subscriptions` row, default Free). The owner holds a **seat pool** and distributes seats by inviting members; limits cover **all** workspaces the owner owns.

- **Limits source of truth:** `src/lib/billing/plans.ts` ‚Äî `PLAN_LIMITS` (free/startup/professional/enterprise ‚Üí `seats`, `agents`, `storageBytes`, `auditDays`, `workspaces`). Mirrors the pricing page. `Infinity` = unlimited. **Seats count the owner too** (Free=2 ‚Üí owner + 1 invitee).
- **Service:** `src/lib/services/billing.ts` (cookie-free, owner-id params) ‚Äî `getOwnerPlan`, `getPlanForWorkspace`, `resolveBillingOwner`, `countSeats` (distinct users across owner's workspaces), `isUserInOwnerPool`, `countAgents` (PAT+OAuth), `getOwnerStorageBytes` (pooled), `getOwnerUsage`, and `check*` helpers returning a `BillingLimitCode | null` (`seatLimitReached`/`agentLimitReached`/`storageLimitReached`/`workspaceLimitReached`).
- **Enforcement (all "block new, keep existing"; platform admins bypass):** seat ‚Üí `inviteToWorkspace` ([auth.ts](src/lib/actions/auth.ts)); workspace cap ‚Üí `createWorkspace` ([workspace.ts](src/lib/actions/workspace.ts)); agent ‚Üí `mintAgentToken` ([agentToken.ts](src/lib/actions/agentToken.ts)) **and** OAuth issuance ([api/oauth/token](src/app/api/oauth/token/route.ts), `authorization_code` grant only); storage ‚Üí [api/upload](src/app/api/upload/route.ts) (413). Re-adding a user already in the pool is free.
- **Demo accounts (`role==='demo'`) can browse pricing but never reach Stripe.** `createCheckoutSession`/`createPortalSession`/`cancelSubscription` ([billing.ts](src/lib/actions/billing.ts)) hard-return `Errors.billingDemoBlocked` for demo. Client-side: `BillingModal`/`PlanPickerModal`/`PricingCtaButton` take an `isDemo` prop (threaded from `currentUser.role`/`session.user.role`) and, instead of checkout, render the shared `DemoBillingNotice` ([src/components/features/DemoBillingNotice.tsx](src/components/features/DemoBillingNotice.tsx)) ‚Äî a friendly "sign in with your own account" panel whose CTA calls `logout()` ‚Üí `/login`. `BillingModal` also hides the Stripe-portal button for demo. i18n: `Billing.demo{Title,Body,Cta}` + `Errors.billingDemoBlocked` (all 6 locales).
- **Billing-owner consistency:** set on `createWorkspace`, seed (`createSeedWorkspace`), first-user bootstrap (`auth.ts` `createUser`), and moved by `transferWorkspaceOwnership`.
- **Stripe:** `src/lib/stripe.ts` (null when `STRIPE_SECRET_KEY` unset), actions in `src/lib/actions/billing.ts` (`createCheckoutSession`, `createPortalSession`, `getMySubscription`, `getMyTier`, `getWorkspaceSeatUsage`), webhook `src/app/api/webhooks/stripe/route.ts` (raw body + signature; whitelisted in `auth.config.ts` + bypasses intl in `proxy.ts`).
  - **Self-healing sync:** every `customer.subscription.*` event calls `syncSubscriptionForCustomer(customerId)` ([src/lib/billing/sync.ts](src/lib/billing/sync.ts)) which **recomputes the effective tier from Stripe's live state** (lists the customer's subscriptions, picks the highest active tier, or Free if none) ‚Äî robust to event ordering AND duplicate subscriptions. Never trust a single event's tier.

Only the first purchase goes through Checkout.
- **Seat distribution & invites:** `inviteToWorkspace` ([auth.ts](src/lib/actions/auth.ts)) adds an existing user directly; for an **unregistered email** it creates a `workspace_invites` row and returns a shareable `/invite/[token]` link. `countSeats` counts members **+ pending invites** (de-duped by email) so invites reserve a seat. ### Performance Rules

- **No query waterfalls:** Fetch independent sources with `Promise.all`.
- **Revalidation:** Call `revalidatePath('/')` only for sidebar-structural mutations (create/delete items, workspace rename/delete). Content saves (`updatePageContent`, `updatePageProperties`) must NOT call revalidatePath.
- **Optimistic UI:** `WorkspaceSidebar` applies mutations to local state immediately; server revalidation is background.
- **SQLite PRAGMAs:** WAL mode, `synchronous=NORMAL`, `foreign_keys=ON`, `cache_size=-20000`, `temp_store=MEMORY` applied at startup in `src/db/index.ts`.
- **Idle Polling Convention:** Automatic background content refreshing via `useWorkspaceEvents()` is configured to only run when the user is inactive (idle) for at least 10 seconds. Active user interactions (keyboard, mouse, touches, scrolling) pause automatic polling to avoid jarring layout/focus resets during data entry.
- **Manual Refresh:** Pages (Databases and Standalone Pages) have a dedicated, localized refresh/sync button in the upper header bar next to the width settings to allow on-demand synchronization when the user is actively working.

### Activation Funnel Analytics

**Goal:** measure the activation funnel **Landing ‚Üí signup ‚Üí first MCP token ‚Üí first agent call**, broken down by channel. Targets: signup‚Üítoken > 40%, token‚Üífirst-call > 50%.

- **Events** (all server-side via [src/lib/analytics/server.ts](src/lib/analytics/server.ts) ‚Äî the middle steps have no browser): `$pageview` on `/` (client, already existed) ‚Üí `signup` ([auth.ts](src/auth.ts) `createUser` event) ‚Üí `mcp_token_created` (PAT: `mintAgentToken` [agentToken.ts](src/lib/actions/agentToken.ts); OAuth: `/api/oauth/token` **authorization_code grant only** ‚Äî refresh is rotation, not a new agent) ‚Üí `agent_call` ([api/mcp/context.ts](src/app/api/mcp/context.ts) `logActivity`, **successful calls only**, fire-and-forget; resolves the token owner via `TokenContext.ownerUserId` set in [api/mcp/route.ts](src/app/api/mcp/route.ts)).
- **Consent (GDPR):** capture respects consent but **never drops a funnel count**. `allowed` ‚áí identified on `userId`; `!allowed` (EU/EEA/UK not-accepted) ‚áí **anonymous** event (random distinctId, PII stripped, `$process_person_profile:false`) ‚Äî aggregate counts survive, nothing personal stored. **admin + demo are never captured** (mirrors `PostHogIdentify`). Request scopes resolve consent from cookie+geo; cookie-less scopes (MCP/OAuth) read `user.analytics_consent`, which the client `ConsentProvider` persists via `setAnalyticsConsent`.
- **Agent-add funnel instrumentation (the middle, between `signup` and `mcp_token_created`):** the coarse funnel hides *where/why* a user fails to connect an agent, so the OAuth/PAT add flow is instrumented step-by-step:
  - **In-app (client, posthog-js, consent auto-respected):** `connect_flow_opened` ‚Üí `connect_editor_selected` ‚Üí `connect_completed` ‚Äî all carry a `source` prop (`onboarding`|`agents_modal`|‚Ä¶) threaded `OnboardingGuide`/`AgentsModal` ‚Üí `ConnectModal` ‚Üí `ConnectFlow`. The gap `connect_editor_selected` ‚Üí `oauth_authorize_viewed` = config friction (editor never kicked off OAuth); `connect_completed` vs `agent_call` = "thinks connected but no real call".
  - **OAuth (server, [api/oauth](src/app/api/oauth/)):** `oauth_authorize_viewed` (consent screen reached ‚Äî [authorize/page.tsx](src/app/[locale]/oauth/authorize/page.tsx)) ‚Üí `oauth_consent_result` (`{result: approved|denied}`, same file's approve/deny actions) ‚Üí on exchange: success keeps `mcp_token_created`, **failure emits `oauth_token_exchange_failed` with `reason`** ([api/oauth/token](src/app/api/oauth/token/route.ts): `missing_params`/`code_not_found`/`code_already_used`/`code_expired`/`*_mismatch`/`pkce_invalid`/**`agent_limit_reached`**). Pre-user failures are anonymous (`captureAnonymous`); user-known ones attribute via `captureForUser`.

### Crash Reporting & Source Maps

Render crashes are caught by the error boundaries (`global-error.tsx` + `[locale]/error.tsx`) and forwarded to PostHog **Error Tracking** as `$exception` events via `src/lib/reportClientError.ts` (`posthog.captureException` + `digest`/`path`/`boundary` props). **Claude Desktop (`.mcpb` bundle)** (`mcpb/`) ‚Äî one-click MCP install for Claude Desktop. `.mcpb` only packages a **local stdio** server, but Remnus's MCP is **remote HTTP** ‚Äî so the bundle ships a thin launcher (`mcpb/server/index.js`) that runs the bundled [`mcp-remote`](https://www.npmjs.com/package/mcp-remote) proxy against `${user_config.server_url}` (default `https://www.remnus.com/api/mcp` ‚Äî **must be the `www` canonical host**: the apex `remnus.com` 307-redirects to `www`, so the OAuth protected-resource metadata reports `www`, and `mcp-remote` fatally rejects a resource-indicator mismatch if handed the apex). `mcp-remote` bridges stdio ‚áÜ the remote Streamable-HTTP server and runs the **OAuth 2.1 + PKCE** browser flow on the first 401 (dynamic client registration against `/api/oauth/*`; the register endpoint already allows `localhost` redirect URIs). The launcher passes `--static-oauth-client-metadata '{"client_name":"Claude"}'` so the registered client is named "Claude" (mcp-remote otherwise registers a generic "MCP CLI Proxy") ‚Äî this makes the connection show the Claude brand icon in `AgentsModal` out of the box. **No token to paste ‚Äî real one-click + OAuth sign-in.**
- Files: `mcpb/manifest.json` (`manifest_version 0.3`, `server.type: "node"`, `user_config.server_url` for self-hosters), `mcpb/server/{index.js,package.json}` (deps: `mcp-remote`), `mcpb/icon.png` (from `public/logo-square-dark.png`), `mcpb/.mcpbignore`.
- Build: `npm run mcpb:build` (= `mcpb:install` + `mcpb:pack` via `npx @anthropic-ai/mcpb`) ‚Üí `mcpb-build/remnus.mcpb` (~1.5 MB; `mcp-remote` + deps bundled so it's self-contained, no `npx` at runtime). Build outputs + `mcpb/server/node_modules/` + `*.pem` are git-ignored.
- **Signing:** `mcpb sign --self-signed` produces a valid PKCS#7 trailer but Claude Desktop still shows an **"unverified publisher"** warning for self-signed (and local `mcpb verify` can't validate it ‚Äî node-forge pkcs7 verify + self-signed not in the OS trust store). **Production trust needs a CA-issued code-signing certificate** (`mcpb sign --cert cert.pem --key key.pem`), best run in release CI (Linux). **Tauri** (`src-tauri/`) ‚Äî Rust shell wrapping a system WebView.
- Dev: `build.devUrl = "http://localhost:3000"` signals CLI to wait; `setup()` hook navigates to `localhost:3000/tauri-app` via `window.eval` (`#[cfg(debug_assertions)]`).
- Prod: loads `https://remnus.com/tauri-app` (set via `app.windows[0].url`).
- Entry point `/tauri-app` sets `localStorage.platform=tauri` and detects OS locale before redirecting to `/app`.
- Features: system tray (single icon, built programmatically ‚Äî **no** `trayIcon` config in `tauri.conf.json`), global shortcuts, notifications, deep-link (`remnus://` scheme).
- **Single instance:** `tauri-plugin-single-instance` (with `deep-link` feature) is registered **as the first plugin** in `lib.rs`. Because the app hides to tray instead of quitting, re-launching the binary would otherwise spawn duplicate processes/windows. The plugin's callback runs in the already-running primary instance ‚Äî it focuses the existing window (`focus_main_window`) and the second process exits. On Windows/Linux the second instance receives `remnus://` deep-link URLs via `argv`, which the callback forwards to `handle_deep_link_url` (shared with the macOS `on_open_url` handler). Deep-link navigation uses the typed `WebviewWindow::navigate(Url)` API (never `eval` string interpolation) to avoid JS code injection from a crafted token.
- **Close to tray:** `CloseRequested` event is intercepted in `lib.rs`; window hides instead of quitting. Tray left-click or "Show Window" menu item restores it; "Quit Remnus" exits.
- **Downloads:** the main window registers a `WebviewWindowBuilder::on_download` handler (`lib.rs`). On `Requested` it redirects the file into the user-chosen download folder (kept in `DownloadConfig` managed state; persisted to `download-dir.txt` in the app config dir, loaded on `setup`) when one is set, keeping the WebView's filename; on `Finished` it emits a `download-finished` event (`{success, path, name}`). The frontend `DownloadToast` ([src/components/features/DownloadToast.tsx](src/components/features/DownloadToast.tsx), mounted in `(app)/layout.tsx`, Tauri-only) listens for it and shows a toast with a "Show in folder" button ‚Üí `reveal_download` command. `reveal_download` tries `tauri-plugin-opener`'s `reveal_item_in_dir` (selects the file) and **falls back to `open_path` on the containing folder** when reveal fails (SHOpenFolderAndSelectItems on Windows can return ERROR_FILE_NOT_FOUND even when the file exists / fail to build an item-id list ‚Äî without the fallback "Show in folder" silently no-op'd). The download-folder picker + WebView zoom controls live in the **Desktop tab of `UserSettingsModal`** ([src/components/features/workspace-settings/DesktopTab.tsx](src/components/features/workspace-settings/DesktopTab.tsx) ‚Äî only shown when running in Tauri; the old standalone `DesktopSettingsModal` + its sidebar Monitor button were removed). Three Rust commands back the picker ‚Äî `get_download_dir` / `pick_download_dir` (native folder dialog via `tauri-plugin-dialog`, Rust-side so no JS capability needed) / `reset_download_dir`. Zoom helpers (`applyDesktopZoom`/`initDesktopZoom`/`getSavedZoom`) live in [src/lib/desktop/zoom.ts](src/lib/desktop/zoom.ts). New Rust dep: `tauri-plugin-dialog`. i18n: `Workspace.download*`/`Workspace.zoom*` keys + `UserSettings.tabDesktop` (all 8 locales).
- **Desktop OAuth flow (polling / device-authorization):** Tauri login view generates a UUID `device_id` ‚Üí opens `<public app URL>/client-login?device_id=<uuid>` in the system browser via `@tauri-apps/plugin-opener` (`window.location.origin` for localhost dev, otherwise `NEXT_PUBLIC_APP_URL`) ‚Üí user logs in (Google or GitHub) ‚Üí browser POSTs to `/api/auth/client-bridge?device_id=<uuid>` which stores a 5-min JWT ‚Üí browser shows "Close this tab" page ‚Üí Tauri WebView polls `/api/auth/client-poll?device_id=<uuid>` every 2 s ‚Üí on `{ ready: true, token }`, WebView navigates to `/api/auth/client-activate?token=‚Ä¶` ‚Üí session cookie set ‚Üí redirect to `/app`.

- Icons: generated from `public/logo-square-dark.png` via `npm run tauri:icon` (after Rust install)

**Capacitor** (`capacitor.config.ts`, `android/`) ‚Äî native WebView wrapper for iOS and Android.
- Loads `https://remnus.com` via `server.url` ‚Äî no static export needed
- Plugins active: `SplashScreen`, `StatusBar`, `PushNotifications`, `Haptics`, `App`, `Keyboard`
- Dark theme colors (`#1d1f23`) set in `android/app/src/main/res/values/colors.xml`
- `android/` is committed to git (native project); `ios/` added on macOS via `npx cap add ios`
- **Requires:** Android Studio (Android) / Xcode on macOS (iOS)
